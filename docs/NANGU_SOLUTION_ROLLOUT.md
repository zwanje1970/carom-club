# 난구해결 해법 편집 — 롤아웃 순서 및 원칙

난구해결사(커뮤니티)의 **해법 작성** 기능은 두께·당점·백/팔로우 스트로크·볼스피드·경로선·곡선 등 **고난도 UI·좌표·저장 흐름**을 포함한다.  
이 영역은 **난구해결사에서 먼저 안정화**한 뒤, 검증된 구성만 난구노트 앱으로 이식한다.

---

## 작업 순서 (고정)

1. **난구해결사 쪽** 기능을 먼저 안정화한다.
2. 버그 패치, **모바일 레이아웃**, **저장 흐름**, **확대/축소·좌표** 문제를 우선 해결한다.
3. 실제 사용 가능한 수준으로 **검증**한다.
4. 그때 안정화된 **로직·UI를 공용 모듈로 분리**하고, 난구노트 앱에 **이식**한다.

---

## 원칙

| 원칙 | 설명 |
|------|------|
| 원본 우선 | 난구해결사에서 원본 기능을 **완성**한다. |
| 검증 후 이식 | 난구노트에는 **검증 완료된 기능만** 가져간다. |
| 단일 수정 지점 | 동일 버그를 **두 군데에서 동시에** 고치는 구조를 피한다. (임시 복제 확대 금지) |

---

## 코드 기준점 (현재)

| 역할 | 위치 (예시) |
|------|-------------|
| 난구해결사(trouble) 해법 편집 | `components/trouble/TroubleSolutionEditor.tsx`, `app/community/trouble/.../solution/new` — **우선 안정화 대상** |
| 커뮤니티 **난구** 게시판 해법 작성 | `components/nangu/NanguSolutionEditor.tsx`, `app/community/nangu/.../solution/new` (마이페이지 노트와 별개 라우트) |
| 공용 저수준 UI | `components/nangu/*` (`SolutionPathEditorFullscreen`, `NanguSolutionPathOverlay` 등) |

**마이페이지 난구노트**에는 위 해법 **풀 에디터**를 아직 이식하지 않는다. 노트 ↔ 난구해결사 **연결 UX**(요청 전송, 연결 링크, 해법 **읽기·오버레이**)만 허용 범위이며, **풀 작성 UI** 이식은 순서 4번(공용 추출 후)에만 진행한다.  
`TroubleSolutionEditor`와 `NanguSolutionEditor`가 겹치면 버그 수정은 **공용 `components/nangu` 레이어**에서 해결해 이중 패치를 피한다.

---

## 완료 판단 (난구해결사 안정화)

- 모바일에서 레이아웃·터치가 일관됨  
- 저장/재진입 시 `solutionData` 손실 없음  
- 줌/팬 후에도 경로·곡선 좌표가 일치함  
- 회귀 시나리오(간단 체크리스트) 통과  

이후에만 **공용 패키지/컴포넌트 분리** 및 난구노트 이식 이슈를 연다.
