# 사용자 화면 구조 분석 및 개선 제안

**사이트 방향:** 당구장 홍보 + 대회 신청 사이트  
**작업 원칙:** 관리자 기능 유지, 사용자 화면 완성도 우선, 밝은 배경 + 카드형 UI + 당구 포인트 컬러, 모바일 친화, DB/API 유지·UI/UX 중심 개선

---

## 1. 현재 사용자 화면 구조

### 1.1 라우트 목록 (관리자 제외)

| 경로 | 역할 | 현재 상태 |
|------|------|-----------|
| `/` | 홈 | 헤더(로고+대회/커뮤니티/로그인/회원가입) + 히어로(사이트명·설명·대회보기/회원가입) + 링크 카드 3개(대회/커뮤니티/마이페이지) + 푸터 |
| `/tournaments` | 대회 목록 | 헤더 동일 + 제목·설명 + 대회 카드 리스트(이름/조직/장소/일시/경기방식). draft 제외 |
| `/tournaments/[id]` | 대회 상세 | 뒤로(목록) + 제목·일시·장소 + **탭**: 대회안내/대회요강/참가신청/참가자명단/시합문의/결과 |
| `/community` | 커뮤니티 | 헤더 + 제목 + 안내 문구만 (실질 콘텐츠 없음) |
| `/login` | 로그인 | 중앙 카드 폼(닉네임/비밀번호) |
| `/signup` | 회원가입 | 중앙 카드 폼(이름/닉네임/연락처/비밀번호/핸디/AVG) |
| `/mypage` | 마이페이지 | 기본정보 + AVG 증빙 업로드. 로그인 필수 |

**당구장 관련:**  
- **사용자용 당구장 라우트 없음.** 관리자만 `/admin/venues`, `/admin/venues/[id]/promo` 로 당구장(Organization type VENUE)·홍보문(promoPublished) 관리. DB에는 조직·slug·홍보 HTML 보관 가능하나, 사용자가 볼 **공개 당구장 목록/상세 페이지는 아직 없음.**

### 1.2 공통 요소

- **테마:** `globals.css` — `--site-bg`, `--site-card`, `--site-text`, `--site-border`, `--site-primary`, `--site-secondary`. SiteThemeStyles로 DB 설정 반영.
- **레이아웃:** 홈/대회목록/커뮤니티는 각자 헤더 포함(로고+nav). 대회 상세는 헤더 없이 max-w-4xl 본문만.
- **인트로:** IntroRoot → 첫 방문 시 인트로 스크린 표시 후 메인 콘텐츠.
- **관리자 플로팅:** AdminFloatButton으로 관리자 전용 진입.

### 1.3 페이지별 상세

- **홈**  
  - 히어로: SiteHeroContent(사이트명, 설명, 대회 보기/회원가입).  
  - 카드 3개: 대회 / 커뮤니티 / 마이페이지.  
  - **갭:** “당구장 홍보” 강조 부족, 당구장 진입점 없음.

- **대회 목록**  
  - 카드 리스트만. 이미지·상태 뱃지·모집 여부 등 없음.  
  - 모바일에서 카드가 단순하고, CTA가 약함.

- **대회 상세**  
  - 탭: 대회안내(description) / 대회요강(outlinePublished) / 참가신청(폼) / 참가자명단(테이블) / 시합문의(안내) / 결과(placeholder).  
  - 참가신청: 로그인 필요, 입금자명·참가요건 동의·참가비 표시.  
  - **갭:** 상단 요약(일시/장소/모집상태/참가비)이 탭 안에 묶여 있음. 모바일에서 탭/테이블 가독성 개선 여지.

- **참가 신청(탭 내)**  
  - TournamentApplyForm: 참가비, 참가요건 HTML, 입금자명, 동의 체크, 제출.  
  - **갭:** 단계 안내·완료 피드백·에러/성공 메시지 스타일 통일 필요.

- **로그인/회원가입**  
  - 단일 카드 폼. next 파라미터(로그인 후 리다이렉트) 등은 있으나, 비주얼은 기본 수준.

- **마이페이지**  
  - 기본정보 + AVG 증빙. 카드형이지만 섹션 구분·강조는 약함.

## 2. “당구장 홍보 + 대회 신청” 기준 갭

- **당구장 노출 부재**  
  - 사용자가 “당구장 소개/오시는 길”을 볼 공개 페이지가 없음.  
  - 관리자가 만든 홍보문(promoPublished)을 노출할 **당구장 목록 + 당구장 상세(예: `/venues`, `/v/[slug]`)** 가 필요.

- **홈 메시지**  
  - 현재: “당구 대회 · 커뮤니티” 위주.  
  - 제안: “당구장 홍보 + 대회 신청”에 맞춰 **당구장 찾기/대회 참가**를 동등하게 강조.

- **대회 목록/상세**  
  - 목록: 카드에 모집 상태·마감일·참가비 등 한눈에 보이면 좋음.  
  - 상세: 상단에 요약 카드(일시/장소/모집/참가비) + 탭 구조 유지하되, 모바일에서 탭·테이블 가독성 개선.

- **참가 신청**  
  - 단계 표시(조건 확인 → 입금자명·동의 → 제출)·성공/실패 메시지·버튼 스타일을 사이트 톤에 맞춤.

- **모바일**  
  - 헤더 네비·카드 그리드·탭·테이블이 이미 반응형이지만, 터치 영역·폰트·여백 등 세부 조정으로 “보기 쉽게” 개선 가능.

- **네이버 지도/이미지/PWA**  
  - 당구장 상세에 지도 연동 시 사용자 이득 큼. 이미지 업로드(로고/홍보)는 관리자 쪽 구현됨. PWA는 아직.

## 3. 우선 손볼 페이지 제안 (우선순위)

아래 순서는 **사이트 방향(당구장 홍보 + 대회 신청)** 과 **노출 빈도·전환(참가 신청)** 을 기준으로 한 제안이다.

---

### 1순위: 홈 화면 개선

- **이유:** 첫인상 + “당구장”과 “대회”를 동시에 보여줄 수 있는 자리.
- **제안 내용:**  
  - 히어로 문구를 “당구장 홍보 + 대회 신청”에 맞게 조정(예: 당구장 소개·대회 참가 강조).  
  - **당구장** 진입 링크 추가(아래 2순위에서 당구장 목록 페이지가 생기면 여기서 연결).  
  - 링크 카드를 **당구장 / 대회 / 마이페이지** 등으로 재구성하고, 밝은 배경 + 카드형 + 당구 포인트 컬러로 통일.  
  - 푸터에 간단 안내(사이트 성격, 문의) 추가 검토.

---

### 2순위: 당구장 안내 페이지 (신규 + 개선)

- **이유:** 방향성의 절반인 “당구장 홍보”가 현재 사용자 화면에 없음.
- **제안 내용:**  
  - **당구장 목록** (`/venues` 또는 `/v`): Organization type VENUE만 조회, 카드 리스트(이름·한줄소개·썸네일 등).  
  - **당구장 상세** (`/v/[slug]`): `promoPublished` HTML 노출, 주소·오시는 길·운영시간 등 메타 정보 있으면 함께 표시.  
  - DB/API는 기존 Organization·admin promo 유지. 공개용 읽기 API 또는 기존 데이터로 페이지만 추가.  
  - 이후 네이버 지도 연동은 당구장 상세에 “오시는 길” 블록으로 추가하는 것이 자연스러움.

---

### 3순위: 대회 목록·상세 페이지 개선

- **이유:** “대회 신청”으로 직결되는 경로.
- **제안 내용:**  
  - **목록:** 카드에 상태 뱃지(모집중/마감/진행중), 일시·참가비 요약, CTA(자세히 보기/참가하기) 등 추가. 카드형·포인트 컬러 유지.  
  - **상세:** 상단에 요약 카드(일시/장소/모집 상태/참가비/참가 신청 버튼) 배치 후, 아래에 기존 탭(안내/요강/참가자/문의/결과) 유지.  
  - 모바일에서 탭은 가로 스크롤 또는 버튼 그룹으로 가독성 개선.

---

### 4순위: 참가 신청 페이지(탭) UX 개선

- **이유:** 전환(참가 완료) 직전 단계.
- **제안 내용:**  
  - 단계 표시(1. 참가비·요건 확인 → 2. 입금자명·동의 → 3. 신청 완료).  
  - 에러/성공 메시지를 카드·배지 스타일로 통일.  
  - 버튼(신청하기/취소)을 사이트 primary/secondary로 통일.  
  - 로그인 유도 문구·링크를 더 눈에 띄게.

---

### 5순위: 모바일 최적화

- **이유:** 당구장·대회 정보는 모바일 접속이 많을 수 있음.
- **제안 내용:**  
  - 전 사용자 페이지 공통: 터치 타겟 크기, 폰트 크기, 카드 여백, 헤더/네비 접기(햄버거) 검토.  
  - 대회 상세 탭·참가자 명단 테이블: 모바일에서 카드형 리스트로 전환하거나 스크롤 안내.  
  - 홈·당구장·대회 목록 그리드: 1열(모바일) → 2~3열(태블릿 이상) 유지하되 갭·패딩 조정.

---

### 6순위: 네이버 지도 / 이미지 업로드 / PWA

- **이유:** 당구장 상세와 사용자 편의를 높이는 부가 기능.
- **제안 내용:**  
  - **네이버 지도:** 당구장 상세(오시는 길)에 연동. 관리자 연동 설정은 이미 있음.  
  - **이미지:** 사용자 화면에서의 업로드(예: 프로필/AVG)는 기존 API 유지, UI만 정리.  
  - **PWA:** manifest + service worker로 설치·오프라인 안내. 홈·대회 목록 등 캐시 전략만 단순하게 적용해도 효과 있음.

---

## 4. 요약 표

| 순위 | 대상 | 작업 성격 | 비고 |
|------|------|-----------|------|
| 1 | 홈 | 문구·링크·카드 구성·푸터 | 당구장 링크는 2순위 페이지 확정 후 연결 |
| 2 | 당구장 안내 | **신규** 목록·상세 페이지 | `/venues`, `/v/[slug]`. DB/API 유지 |
| 3 | 대회 목록/상세 | 카드·요약·탭·모바일 | 기존 데이터로 UI만 개선 |
| 4 | 참가 신청(탭) | 단계·메시지·버튼 스타일 | TournamentApplyForm 등 |
| 5 | 모바일 | 공통 터치·타이포·레이아웃 | 전 페이지 |
| 6 | 지도/이미지/PWA | 당구장 지도, PWA manifest 등 | 부가 기능 |

---

## 5. 다음 단계 제안

1. **1순위(홈)** 부터 적용: 히어로·카드·네비 문구만 바꿔도 “당구장 홍보 + 대회 신청” 메시지가 명확해짐.  
2. **2순위(당구장)** 에서 공개 당구장 목록/상세 라우트와 데이터 조회 방식(기존 Organization + promo)을 정한 뒤, 홈에 “당구장” 링크를 넣음.  
3. 이후 3→4→5→6 순으로, DB/API 건드리지 않고 UI/UX만 단계 적용.

이 순서로 진행하면 관리자 기능은 그대로 두면서, 사용자 화면을 “당구장 홍보 + 대회 신청” 사이트에 맞게 정리할 수 있다.
